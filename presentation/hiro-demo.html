Deny Mode

Signatures in a Safe Environment


AI orchestrates. You sign.


  • Today: sign in the Pillar web app
  • Tomorrow: sign in the Pillar mobile app
  • Your keys never leave the secure environment
This is by design - we call it Deny Mode. The AI orchestrates the action, but YOU sign. Your passkey stays in a safe environment - the browser or mobile app. Keys never touch the AI. This isn't a limitation, it's a feature.

The Flow


  +------------------+     +------------------+     +------------------+
  |  Claude / MCP    |---->|  Pillar Backend  |---->|   Pillar App     |
  |  (orchestrate)   |     |  (create intent) |     |  (sign safely)   |
  +------------------+     +------------------+     +------------------+
          |                                               |
          |<-------------- confirmed txId ----------------|

Chat with Claude anywhere (Desktop, WhatsApp, terminal...)
Switch to Pillar app for final signature

Here's how it works: Claude sends the intent to our backend. Backend creates a pending operation. Your browser opens with that operation. You sign with your passkey. Transaction goes on-chain. Claude gets the confirmed txId back. You can chat from anywhere - desktop, WhatsApp, terminal - and just switch to the app to sign.

Architecture


+---------------+    +---------------+    +---------------+
|    Claude     |--->|  Pillar BE    |--->|  Pillar App   |
|  (anywhere)   |    |   (Nitro)     |    | (safe signer) |
+---------------+    +---------------+    +---------------+
                           |
                    +------+------+
                    |  Stacks +   |
                    |  sBTC + Zest|
                    +-------------+
Quick architecture overview. Claude can run anywhere. Talks to our Nitro backend. Backend manages operations and talks to the app for signing. Everything settles on Stacks with sBTC and Zest Protocol.
